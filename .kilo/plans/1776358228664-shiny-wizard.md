# Plan de Auditoría Completa del Proyecto LoneWriter

## Resumen Ejecutivo
Se ha realizado una auditoría completa del proyecto LoneWriter (v1.5.0), identificando varios problemas técnicos que deben corregirse.

---

## Hallazgos Críticos (Deben corregirse)

### 1. ERROR: Versiones de Dexie fuera de orden secuencial
**Ubicación:** `src/db/database.js:6-76`
**Problema:** Las versiones de la base de datos no están en orden numérico:
- v2, v3, v4, **v6**, **v5**, v7, v8, v9, v10, v11, v12
- Las versiones 5 y 6 están intercambiadas
**Impacto:** CRÍTICO - Puede causar fallos en la migración de la base de datos IndexedDB existente
**Corrección sugerida:** Intercambiar las definiciones de v5 y v6 para que queden: v2, v3, v4, v5, v6, v7...

### 2. ERROR: Bug de variable en getDragLabel()
**Ubicación:** `src/views/Editor.jsx:947`
**Problema:** Error de variable inconsistente:
```javascript
const sc = (ch.scenes || []).find(s => `scene-${s.id}` === idStr);
```
Debería ser:
```javascript
const sc = (ch.scenes || []).find(scene => `scene-${scene.id}` === idStr);
```
O simplemente:
```javascript
const sc = (ch.scenes || []).find(s => `scene-${s.id}` === idStr);
```
El problema es que la variable se llama `sc` pero luego en el template string usa `s.id`.
**Corrección sugerida:** Cambiar a `scene-${scene.id}` usando nombre consistente

## Plan de Corrección

### 1. ERROR: Versiones Dexie fuera de orden (CRÍTICO)
**Archivo:** `src/db/database.js`
**Acción:** Reordenar las definiciones de v5 y v6
**Riesgo para usuarios:** MÍNIMO - Solo afecta a usuarios nuevos o tras borrar datos del navegador
**Método seguro:** Cambiar solo el ORDEN de las definiciones, NO los números de versión

Código actual (incorrecto):
```javascript
db.version(6).stores({   // ← Definido ANTES que v5
  oracleEntries: '++id, novelId, sceneId, createdAt, isCorrected'
});
db.version(5).stores({   // ← Definido DESPUÉS que v6
  characters: '++id, novelId, name, role, ignoredForOracle',
  ...
});
```

Código correcto (intercambiar orden):
```javascript
db.version(5).stores({   // ← primero v5
  characters: '++id, novelId, name, role, ignoredForOracle',
  ...
});
db.version(6).stores({   // ← luego v6
  oracleEntries: '++id, novelId, sceneId, createdAt, isCorrected'
});
```
