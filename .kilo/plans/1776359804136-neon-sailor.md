# Plan: LoneWriter v1.7

## Feature 1: Rewrite with Context (Previous Paragraph)

### Goal
Add automatic context injection to the Rewrite tool, providing the previous paragraph(s) to maintain narrative fluency.

### Implementation

**1.1 Modify `src/services/aiService.js`**
- Update the `rewrite` function to accept a new `previousContext` parameter
- Inject `[CONTEXTO PREVIO]` tag into the prompt when context is provided
- Label localized as `[CONTEXTO PREVIO]:` (ES) / `[PREVIOUS CONTEXT]:` (EN)
- Add directive: "Adapta la fluidez del texto nuevo sin reescribir el texto antiguo" / "Adapt the flow of the new text without rewriting the old text"

**1.2 Modify `src/components/AIPanel.jsx`**
- Add function to retrieve previous paragraph(s) from the editor content
- Pass the context to `AIService.rewrite()` when calling the rewrite function
- The context should be extracted from the RichEditor's HTML content

**1.3 Extract Paragraph Logic**
- Parse the editor content to find paragraph(s) before the selected text
- **Token/Word limit approach** (user preference):
  - Limit context by number of tokens/words, not paragraphs
  - This avoids extremely short paragraphs (just a word) and extremely long ones
  - Recommended: ~100-150 words for previous context (~75 tokens)
  - For RAG mode: also limit to ~200-300 words total
- Two modes:
  1. **Simple mode**: Previous ~100 words before selection
  2. **RAG mode**: Semantic search for relevant context (more complex, uses existing RAG service)

## Feature 2: Heuristic Active Entity Suggestion

### Goal
Automatically suggest which compendium entities should be active for the Oracle, based on keyword matching + anaphora resolution to avoid "pronoun fatigue" and indirect references.

### ⚠️ Memory Concern - Analysis
- **Mismo patrón existente**: LoneWriter ya carga TODAS las entidades del compendio en memoria para el Oracle y MPC
- **Estimación**: 50-200 entidades × 1-2KB = ~100-400KB por novela (típico)
- **Conclusión**: El patrón ya funciona en dispositivos móviles, no se necesita optimización adicional
- **Mismo debounce**: 500ms-1s (igual que MPC)

### Implementation

**2.1 Create new service `src/services/heuristicEntitySuggester.js`**

**2.1.1 Anaphora Resolution System (Local, zero cost)**
- Track "last mentioned entity" per scene/chapter (in-memory, no DB persistence needed)
- Pronoun mapping dictionary (ES/EN):
  - ES: él, ella, ellos, ellas, este, esta, estos, estas, ese, esa, esos, esas, lo, le, se, su, sus
  - EN: he, she, it, they, this, that, these, those, him, her, them, his, hers, its, their
- When a pronoun is detected in recent text (last ~100 tokens), resolve to last mentioned entity of matching gender/number
- Store in context window: `{ entityId, entityTable, lastMentionTokenPosition }`

**2.1.2 Keyword Matching (Local, zero cost)**
- Scan text against entity names/nicknames/keywords using regex
- **No API calls** - completely local operation
- Scoring system:
  - Exact name match: +10 points
  - Nickname match: +5 points  
  - Keyword match: +3 points
  - Anaphora resolved match: +8 points (pronoun → entity)
  - Recency bonus: +2 points if found in last 100 tokens
- Return ranked list of suggested entities with scores and match type

**2.2 Database Schema**
- **Sin cambios en DB**: El sistema funciona 100% en memoria (in-memory)
- No se necesita nueva tabla en Dexie - reduce complejidad y overhead
- Las sugerencias se mantienen en el estado de React durante la sesión
- Al cambiar de escena/novela, se reinicia el contexto de anáforas

**2.3 Create UI Component `src/components/EntitySuggestionPanel.jsx`**
- Displays suggested entities as chips/badges
- Shows glow effect on detected entities
- Visual distinction for match types (icon/badge):
  - 🔵 Direct match (name/keyword)
  - 🟣 Anaphora match (pronoun resolved)
- "One-click" to activate the entity (toggle the Zap)
- Does NOT auto-activate - user retains full control
- Option to "Apply all suggestions" (bulk activate)

**2.4 Integrate into Editor or Compendium view**
- Add debounced scanning (500ms-1s) on text change in the editor
- Run in background Web Worker to avoid blocking UI
- Show subtle indicator (💡 icon) in toolbar when entities are detected in current scene
- Clicking indicator opens suggestion panel

**2.5 Web Worker Integration (Zero Cost)**
- Create lightweight worker for keyword scanning + anaphora resolution
- **No external API calls** - purely local regex/string operations
- Ensure zero latency on the main thread
- Worker receives: text + entity data (from Dexie), returns: ranked suggestions with anaphora resolutions
- Communication via postMessage (same pattern as existing ragWorker)

**2.6 Anaphora Resolution Edge Cases**
- Ambiguous pronouns (e.g., "it" could be object or animal): default to last entity of any type
- Multiple entities of same type in recent context: prefer most recent
- First person pronouns ("yo", "I"): ignore (POV character is already tracked)
- Possessive pronouns ("su", "his"): resolve to entity's owner if known

## Phase 2: Bug Fixes & Improvements

### Issues Reported by User

**1. Feature 1: Missing debug log and no toggle control**

Problem: User cannot find `[CONTEXTO PREVIO]` in console logs, and there's no option to enable/disable context injection.

**Fixes needed:**
- Add explicit `console.log('[Rewrite] Sending previousContext:', previousContext)` in AIPanel.jsx when calling rewrite
- Add a toggle/checkbox in the Rewrite tab to enable/disable context injection
  - Label: "Incluir contexto anterior" (ES) / "Include previous context" (EN)
  - Default: enabled (true)
  - Location: In the Rewrite tab, near the goal buttons

**2. Feature 2: Inconsistent detection + Cross-novel contamination**

Problem: 
- Suggestions don't always detect entities (inconsistent)
- When multiple novels are open, suggestions mix entities from different compendiums

**Root causes identified:**
1. **Caching bug**: `entityDataCacheRef.current` is not cleared when switching novels
2. **Debounce too long**: 800ms may miss fast typing detections
3. **Token matching issues**: Current logic requires exact token match, which fails with partial matches

**Fixes needed:**

**2.1 Fix cross-novel contamination**
- In AIContext.jsx, when `activeNovelId` changes, clear `entityDataCacheRef.current` and `entitySuggesterRef.current.resetContext()`
- Add cleanup in the `initEntitySuggester` effect when novel changes

**2.2 Improve detection reliability**
- Reduce debounce from 800ms to 500ms
- Add case-insensitive matching with looser tokenization
- Add fuzzy matching for compound names (e.g., "Puerto de Valdris" should match "puerto" OR "valdris")
- Add logging: `console.log('[EntitySuggester] Scanned, found:', suggestions.length, 'entities')`

**2.3 Add visibility toggle**
- Add "Mostrar sugerencias" / "Show suggestions" toggle in Editor toolbar
- Currently shows badge only when suggestions exist - make it always visible but dimmed when empty

**2.4 Add debug console logs**
- Log entity data loaded: `[EntitySuggester] Loaded ${totalEntities} entities for novel ${novelId}`
- Log scan results: `[EntitySuggester] Scan complete: ${suggestions.length} suggestions`
